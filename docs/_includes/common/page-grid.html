Setting up a page grid requires two key elements: a containing element, and the grid element itself.

The grid element sets up the columns and gutters on which its child elements can be positioned.

The container takes care of positioning the grid element in relation to the page (typically center aligned) and adds outside margins.

Any content that is off-grid, a full bleed banner for example, should be placed outside of these two elements.

Grid areas are definable with [CSS Grid syntax](https://developer.mozilla.org/en-US/docs/Web/CSS/grid-area). The following CSS shows the grid layout of the Skin website itself.

{% highlight css %} @media only screen and (min-width: 320px){ .site-nav { display: none; } main { grid-area: ~"1 / 1 / span 1 / span 8"; } footer { grid-area: ~"2 / 1 / span 1 / span 8"; } } @media only screen and (min-width: 600px) { .site-nav { grid-area: ~"1 / 1 / span 1 / span 3"; } main { grid-area: ~"1 / 4 / span 1 / span 13"; } footer { grid-area: ~"2 / 1 / span 1 / span 16"; } } @media only screen and (min-width: 968px) { .site-nav { grid-area: ~"1 / 1 / span 1 / span 2"; } main { grid-area: ~"1 / 3 / span 1 / span 14"; } } {% endhighlight %}
