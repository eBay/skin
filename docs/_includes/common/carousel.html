@ebay/skin/carousel

Carousels group together a list of similar items into a horizontal viewport of fixed height. Items may be entirely static (e.g. an image), entirely interactive (e.g. button) or contain a mix of static and interactive elements (e.g. a heading, image and link).

Items may be grouped and paginated via discrete slides, or left ungrouped and paginated in an individual manner.

NOTE: No JavaScript behaviour has been implemented in the carousel examples, we demonstrate the CSS aspect only. For detailed behaviour requirements, please visit the eBay MIND Pattern for Carousel.

Filmstrip Carousel

Any carousel showing more than one item per slide is known as a filmstrip carousel. This is analogous to looking at a strip of photographic film. By default, most carousels fall into this category.

Discrete Filmstrip Carousel

When grouping filmstrip items into discrete slides (4 per slide in the example below), add the carousel--slides modifier.

A discrete carousel requires an ARIA live-region to convey slide updates to assistive technology.

Discrete Filmstrip Carousel with Peek

For discrete filmstrip carousels on touch devices, the carousel paddles will be shown by default when there is no peek (as in the previous example).

A peek can be created, if desired, simply by adjusting item widths and applying the modifier carousel--peek.

Slideshow Carousel

A discrete carousel showing exactly one item per slide is known as a slideshow carousel. This is analogous to an old fashioned slide projector, viewing one image at a time.

The carousel--slides modifier is required.

A slideshow carousel has no peek, therefore all list items should be set to 100% width.
