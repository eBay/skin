The expressive loader can replace our traditional spinner to feel faster during periods of longer loading. To portray responsive and quicker loads, the expressive loader's design uses visual momentum to indicate progression across refreshing, page loading and user actions. It consists of an animated progress bar in various colors and an optional set of messages that are cycled into view.

The expressive loader uses Javascript to cycle through messages and dynamically add new progress bar pieces with random widths. It also renders different experiences depending on the prefers-reduced-motion media query. The following code examples represent snapshots of the expressive loader at any given time. You may want to toggle your reduced motion settings to view all the examples.

Progress bar animation

Use Javascript to add lines that scale up from 0% to 10 - 40% of the width of the container. When a line is finished scaling in, add a new line.

  1. Create a new element with these classes:
    • expressive-loader__line, base class for lines
    • expressive-loader__line-${colorIndex}, where ${colorIndex} is a number 1
    • expressive-loader__line--scaling, which will scale the line's width up
      • through 4. This sets the line's color.
  2. Randomly assign the element a width between 10% and 40%.
  3. Add an event listener for the element's animationend, which triggers the creation of another line and removal of the oldest line.
  4. Prepend the line to the .expressive-loader__lines` container.

Base Expressive Loader with Reduced Motion

When @media (prefers-reduced-motion: reduce), e.g. the user prefers reduced motion, the lines still animate. Instead of growing from 0% to a random size, they stay a fixed size and scroll slowly across the screen. The initial markup is similar to the non-reduced motion implementation, but includes 4 sets of lines instead of one. The first two sets contain lines of varying widths, and the last two sets duplicate the first ones to achieve a smooth marquee effect. No Javascript is needed to animate the reduced motion expressive loader.

Message animation

If there are multiple messages to display, cycle them in and out of view using Javascript. To cycle a message into view:

  1. Animate in the message:
    1. Put the message content in the aria-hidden element with class expressive-loader__message. This element is used just for the visual message animation; it is not read to screen readers.
    2. Add the expressive-loader__message--in class to animate that message into view.
  2. After the message animates in:
    1. Put the message content in the status element with class expressive-loader__message. This will read the message aloud to screen reader users.
    2. Remove the expressive-loader__message--in class from the aria-hidden element.
  3. After a fixed amount of time, add the expressive_loader__message--out class to the current message to animate it out of view. This can happen concurrently with the next message animating in.
